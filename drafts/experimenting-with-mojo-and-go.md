# Experimenting with Mojo (and Go)

Mojo is a programming language that bridges Python with some Rust sensibilities. I discovered it just a couple weeks back as I was enormously frustrated with trying to fully digest Rust, my 3rd attempt to really get comfortable with it. My problem now isn’t so much with the syntax (such as borrowing/ownership, or the lack of nulls) but the extreme wordiness of everything, which I find almost makes it unreadable to an extent at a glance. Yet, I can’t stop thinking about Rust, I know what it is doing is quite genius, but it doesn’t mean it is pleasant.

On the other hand, I have been experimenting with Python, I absolutely adore its reasability but as someone from a pretty heavy TS and C# background its lack of types really made me feel naked and uncomfortable everytime I wrote code in it. I loved how my code look, but I hate how dirty I felt writing it without something to scream at me like a picky Rust compiler.

In walks Mojo.

It uses mainly the syntax for Python, infact it is so close it almost looks like Python through no accident of the developers. You can even use some Python along side it and the main installation instructions now have you installing mojo through Python packages.

I also had an unrelated run in with Go. Oddly, I was just about ready to abandon my Rust efforts for Go when I discovered Mojo only shortly after beginning a Go project just to decide that Mojo had actually won my heart before Go had a true chance to do so. I still love the brevity and simplicity of the small Go syntax overhead. It is very very minimal both in appearence and in language features, but as a result, Go felt more to me like an advanced and performant scripting language than a real language I would actually want to use for a large project.

I think I am at the point now where:

**TS** — If I want to design a UI for something or have a more web-centric API. I have loved Vue and Mithril in the past, but I think at this point TS = React for me. React-Jsx syntax and its type-reinforcing ability is what gives this language the greatest value.

**C#** — If I want a data heavy application that focuses on large structures best represented relationally — for me (personally) C# and OOP is something I will always adore but I will admit I am more often than not looking towards procedural approaches to code design as I have found it just much more pleasant to read (more so than even functional first-class-member patterns). I think heavier APIs is where this shines.

**GO**- If I want to write some quick code to automate something, or launch tasks, especially in parallel. This also would be the language I would choose if no one I was doing code with knew any language — its middle ground between many languages, and ultra lean syntax makes learning it extremely simple.

**Rust** — If I want to have tiny modules with great performance (like WASM) to limit Rust’s annoyance while maximizing and focusing its power where it is needed. I have to ask myself “is the pain of writing this in Rust worth its incredibly performance and security”

**Mojo** — My main go to catch all language going forward (if none of the above applies). Likely replacing Python for me the way TS replaced JS.

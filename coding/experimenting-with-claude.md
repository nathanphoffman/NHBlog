# Experimenting with Claude

![](/images/0-pp_a2Qn5YY8ReQ5y.png)

I began manually creating a [weather terminal app](https://github.com/nathanphoffman/Weather) last year in Typescript, it simply pulls in weather data from NOAA and greatly simplifies the weather data by showing three hour intervals by combining multiple factors into a real-feel temperature, a composite of precipitation, clouds, wind, etc (into “storm rating”) and letter and emoji flags for relevant info like wind, humidity, and freezing temperatures. The idea was to quickly see what the weather was for a week at a glance.

Just a few weeks back I started to put the code into a [NextJS web app](https://github.com/nathanphoffman/WeatherSite) so I could see the weather from anywhere on a personal site, and I realized it might be a good project to try out Claude which I have been hearing about pretty much everywhere. It was an interesting experiment as I also had less knowledge of Next but have plenty of knowledge of Typescript and some React experience so it was a way to both see how useful AI could be as a teaching tool but also how good it was at the things I knew well.

You can find the current site here [weather.nathanhoffman.me](http://weather.nathanhoffman.me)

I will say my overall experience has been quite positive, but it hasn’t been entirely perfect either. I will highlight a few takeaways I have had that highlight both my positives and negatives with this process:

**The NOAA API:** At one point the agent suggested I rewire the data from a scraper into NOAA’s public API. I hadn’t even asked it this and I had no idea they had a public facing API and had been scraping their HTML since I started using the terminal app. I was skeptical at first, and I allowed it to wire it up all on its own. Shockingly, the data worked right out of the box, it was able to figure out the API structure on its own and I saw no issues. But when I went to review the code, it wasn’t all perfect. Rather than replace some of the scraper logic it actually formatted the data with additional conversion code to make it more like the display friendly format the scraper took in, and then passed it into the original scraper scaffolding which effectively removed this formatting data anyway. It also bypassed a candidate-style domain model I had where data would be validated in a more structured way.

**The Charts**: The UI for the charts was effectively vibe coded, I let AI do whatever it wanted to render that data, and to my shock it looked extremely good visually and in general the code was mostly good. Going back through it I had to restructure some of it though like separating out the extremely large main chart component into several smaller ones that broke up AxisLabels, ThresholdLines (the horizontal dashed lines), simplified log-scales (it had both an enable flag and a log-strength there was little point for both) and break out the Line and MultiLine graphs into their own components.

**Removing Comments**: I would frequently post comments in the code-base that would then suddenly go missing as previous changes made by Claude, sometimes totally unrelated, would remove them as if they were inconsequential to the project.

**Component Folder Structure**: I generally like deeply nesting component structures within folders to follow the way in which it is used in React/Next so children are subdirectoried within parents. However, it isn’t worth doing this if a subcomponent is extremely small and has no children of its own and I sometimes keep these with their parent on the same folder level. What was interesting to me, is Claude rarely made new directories for components when I had it add them. It could be because my occasional exceptions to my own rule confused it, but I thought it was strange it never proposed or added new directories following my structure or prompted me with questions.

**Other UI Stuff:** In general the AI did a tremendous job with tailwind, I am not the most knowledgeable with it but have used it in the past, but everytime I questioned its decisions and did some Googling of my own it appeared to have made the right call. It also at one point came up with a somewhat obvious solution I was missing — which was to add an invisible layer to obscure the credits link to Simple Maps to prevent accidental clicks while using the dropdown, it did this very elegantly with just a couple tailwind classes. Unfortunately, the one issue I did see with CSS is it randomly removed a font I was using and then did not reapply it with Tailwind, it was one of the oddest hallucinations I have seen thus far as I did not request it.

**Validation**: As part of my model validation with my candidates I have a series of function checks like “isGreaterThan0” that get passed into arrays that define each property incoming from the API. This existed early on before I used the AI, but interestingly while the AI ignored these at first, it eventually started improving them for me. It realized, for example, that I had missed a few checks like making sure wind speed was non-negative, and it pointed out that I had messed up my check that temperature should be greater than absolute zero. It went further and suggested I make it a “sensible temperature” as it is unlikely the Earth would ever go that low in temp.

**Unit Tests**: It was extremely good at creating unit tests for my validators, partly because they are well structured and named so it likely had a lot of context to go on, but weirdly it didn’t suggest unit tests for anything else and never since has suggested unit testing anything else in my code base (even the agent I added explicitly to look for unit tests to add)

There are plenty more examples, but I think this gets the point across I found it a generally very useful tool but also not without faults. I have found its usefulness almost more in the back and forth I can have with it, similar to the back and forth one can have in pair programming where I can bounce ideas off of it and it ideas off of me.

I have since changed my workflow with Claude and found using agents with well defined markdown files greatly improves my results. I have some thoughts on how to best use this optimally which I might post about in the future, but I don’t want every post to be about AI so I will probably do another pure coding post next time!
